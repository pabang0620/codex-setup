# source-command-learn

Use this skill when the user asks to run the migrated source command `/learn`.

## Command Template

# /learn - 재사용 가능한 패턴 추출

현재 세션을 분석하고 스킬로 저장할 가치가 있는 패턴을 추출합니다.

## 트리거

간단하지 않은 문제를 해결했을 때 세션 중 언제든지 `/learn`을 실행합니다.

## 추출할 내용

다음을 찾습니다:

1. **에러 해결 패턴**
   - 어떤 에러가 발생했는가?
   - 근본 원인은 무엇이었는가?
   - 무엇이 수정했는가?
   - 유사한 에러에 재사용 가능한가?

2. **디버깅 기법**
   - 명확하지 않은 디버깅 단계
   - 효과가 있었던 도구 조합
   - 진단 패턴

3. **해결 방법**
   - 라이브러리 특성
   - API 제한사항
   - 버전별 수정사항

4. **프로젝트 특화 패턴**
   - 발견된 코드베이스 컨벤션
   - 내려진 아키텍처 결정
   - 통합 패턴

## 출력 형식

`.codex/skills/learned/[pattern-name].md`에 스킬 파일 생성:

```markdown
# [설명적인 패턴 이름]

**추출일:** [날짜]
**컨텍스트:** [적용 시기에 대한 간략한 설명]

## 문제
[해결하는 문제 - 구체적으로]

## 해결책
[패턴/기법/해결 방법]

## 예시
[해당되는 경우 코드 예시]

## 언제 사용하나
[트리거 조건 - 이 스킬을 활성화해야 하는 경우]
```

## 프로세스

1. 추출 가능한 패턴에 대해 세션 검토
2. 가장 가치 있고 재사용 가능한 통찰력 식별
3. 스킬 파일 초안 작성
4. 저장 전 사용자에게 확인 요청
5. `.codex/skills/learned/`에 저장

## 참고사항

- 사소한 수정은 추출하지 않음 (오타, 간단한 문법 에러)
- 일회성 이슈는 추출하지 않음 (특정 API 장애 등)
- 향후 세션에서 시간을 절약할 패턴에 집중
- 스킬을 집중적으로 유지 - 스킬당 하나의 패턴
